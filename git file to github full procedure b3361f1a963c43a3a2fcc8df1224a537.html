执行命令 git submodule update --init --recursive 报错，

fatal: 不是一个 git 仓库（或者任何父目录） 

解决办法：

先执行 git init 即可

安裝完git 要做的第一件事情

git config # 只需要做一次就好

第二件事情：

git init ： 將所在目錄做git 初始化

如果要把目錄交給git保管，那在目錄就要先做git init

git status : 確認目前git對於每個檔案的狀態

git add : 將檔案放入git 內 ，準備被git 保管 git add .

git commit ： 輸入註解，將準備被git保管的檔案確認送出 git commit -m “enter the comment”

ls -l 查看目前目錄的所有檔案

full procedure for upload
1. create a new remote node
2. create a new folder in local
# 如果要移動目錄，必須一層一層的cd
#cd .. 回到上層目錄
3. cd to working directory
4. git init
5. git remote add origin 'paste the repository http url'
# origin 只是遠端節點的代名詞
6. git add .
# 轉移至 stage(index) ，也就是緩衝區  準備被commit
7. git commit -m "enter the comment for this commit"
8. git push -u origin master 
# master 是其中一個branch 可以隨意更換
# push master this branch to origin which is the remote node on github
